# はじめに
　本日高校を卒業する。そこで、本日から４記事に分けて、これまでの学業的な経験を振り返ってみようと思う。
一番最初は中学生の頃で、私が情報の道を志したきっかけから、どのように能力を身につけていったかという部分がわかる。
成功したことも失敗したことも、記憶を頼りにできるだけ正直かつ正確に書くつもりである。
他人の経験談に興味がある人は数少ないかと思うが、それでも誰かに少しでも影響を与えることができれば幸いである。

#　PCとMicroBit
小学生の終わり頃、親戚からノートパソコンとMicro:bitを購入していただいたことが、情報技術に関心をもつ契機となった。
元来、私は工作や絵画などの制作活動をしていたこともあり、プログラミングによって自分のアイデアを形にできる可能性に関心を抱いていた。

当初はMicro:bitの説明書兼チュートリアルを用いて基本的な操作を学習した。
しかし、入門キットで実現できる内容には制約があり、より自由度の高い制作環境を求めるようになった。
そこで、以前テレビで紹介されていたScratchに関心を持ち、保護者の助言を受けながら制作を開始した。
当時の私はブラウザの基本的な操作すら十分に理解していなかったため、コンピュータ活用の基礎を学習する段階にあったと言える。

# Scratch
Scratchでの制作活動においては、主としてYoutubeの解説動画や入門書を参考にゲーム制作を行なった。
具体的には、シューティングゲーム、ターン制RPG,横スクロール型アクションゲームなどの作品を制作した。
これらを制作する過程において、変数の概念や関数（Scratchにおける定義ブロック）を用いた処理構造、
並びに変数を利用したアニメーション制御などプログラミングの基礎概念を段階的に習得した。
特に、状態管理を伴う処理やフラグ制御などには困難を感じたが、試行錯誤を繰り返し理解を深めることができた。

#　Unity
Scratchに慣れてくるにつれ、より高度なゲーム開発環境への関心が高まった。
そこで、物理エンジンであるUnityの使用を開始した。
Unityの導入に関しては保護者の支援を受けながら環境構築を進めていった。

Unityの使用にあたって、私は初めてC#言語によるテキストのプログラミングに触れることになった。
当初は書籍のサンプルコードを書き写す形で学習を進めたが、周囲にプログラミング経験者がいない環境であったため、
エラーが発生した際に自ら原因を調査する必要があった。
そのため、コード検証、エラーメッセージの解析、インターネット上の情報を用いて問題解決を行う
習慣が形成された。

また、作成したゲームは具体的に、３Dアクション、FPSゲーム、サンタになって警察から逃げながらプレゼントを届けるゲームである。
これ以外にも、テスト対策用にクイズを自分で作成し解くことができる暗記アプリを作成した。
また、Unityでの成果を親戚に伝えるために、HTMLとCSSで作成内容の解説記事を作った。

# プレゼン
中学2年の頃、中学でGIGAスクール構想の一環として、生徒に一人一台のタブレット端末が配布された。
当時、多くの生徒がコンピュータの操作に不慣れであったが、私は一定期間コンピュータを使用していたため、
周囲の生徒に操作方法を説明したり、不具合の解決を支援したりする機会が多くなった。
その結果、コンピュータの技能について周囲から一定の信頼を得るようになった。

また、学校の授業の一環として、SDGsに関する課題を一つ選択し、その解決策を提案する活動が行われた。
この活動では、アイデアをプレゼンテーションし、クラス内選考及び学年選考の投票によって決定される形式であった。

私の所属する班では、クラス選考においてスライドの作成およびプレゼンテーションの担当をし、
学年選考では動画編集を担当した。結果として、両選考を通過し、最終的に班の提案が採用された。

さらにこの経験で更なる信頼を得て、委員会のスライド制作も依頼された。
一方で、授業内の発表においては、データを視覚的に提示する試みとして、数値の大小を円の面積で示す図や、
ベン図を用いた説明を行なった際には、十分な理解を得ることができなかった。
この経験から、情報の視覚化だけでなく、聴衆の知識レベルを考慮することの重要性を認識した。

#　トラブルシューティング
当時、教員および生徒の多くはコンピュータの操作を十分に習得していなかった。そのため、
私は不具合の原因調査や操作方法の説明などを行う機会が頻繁にあった。

例として、授業中に別クラスの教員から呼ばれ、ディスプレイモードの変更方法を説明したことがある。また、
教員から授業で使用する簡単なプログラムの作成方法について相談を受けることもあった。
こうした経験を通して、技術的な問題に対する対応能力を実践的に身につけるとともに、周囲から信頼される立場を形成した。


# 志望校を見つけてから
中学2年生の頃、IT関連職種を志向する教育課程として情報科学科を設置している高校を知った。
私はそれまでプログラミングについて議論できる同世代の友人や指導者が周囲に存在しないことに物足りなさを感じていたため、
その高校を進学目標とした。
それ以前の私は、授業への参加態度や学習習慣はあまりよいとは言えなかった。授業中の居眠り、宿題の未提出、遅刻などが常態化しており、自身でも高校進学は困難であると認識していた。
そのため、高校進学ができなかった場合を想定し、基本情報技術者試験の学習を開始していた時期だった。
しかし、明確な目標を設定したことにより、学習態度は大きく変化した。授業を積極的に受けるようになり、定期試験でも高得点をとった。
その結果、教員からも学習態度の変化について賞賛されたこともあった。

その後、塾に通い始め、数学を軸に高校入試に向けた学力の向上を目指した。
志望校の入試難易度自体は極端に高いものではなかったが、それまでの内申点を考慮すると、
入試本番で十分な得点を確保する必要があった。また、当該高校では面接の配点が比較的高かったため、
文章構成、話し方、マナーなどについて教員の指導をうけながら対策を行なった。
その結果、志望していた高校に合格し、進学することとなった。

# まとめ
コンピュータを入手してから高校入学に至るまでの期間は、私が情報分野へ進むための基盤を形成した時期であったと言える。

特に、ゲーム開発の経験は、プログラミングだけでなく、３D表現、グラフィック設計、ユーザーインターフェース設計など
多岐にわたる領域を横断する知識を必要とした。
したがって、各分野とそれを統合しながら成果物を作成する能力を習得したと言える。

さらに、プレゼンテーションやトラブルシューティングの経験を通して、技術内容を他人に説明する能力や、
問題の原因を調査し、自分で調べ自分で解決する能力を実践を通して身につけた。
また、志望校を目標として学習に取り組んだことにより、数学および英語といった基礎学力も強化され、
高校以降の学習の基盤となった。

以上の経験を通して、この期間は情報分野に進むきっかけを形成するとともに、その基盤を獲得した時期であったと思う。
