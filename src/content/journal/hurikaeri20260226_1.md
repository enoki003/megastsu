# はじめに
本記事はこれまでを振り返るうち、高校１年生を中心に振り返る。基本情報レベルの知識を蓄え、さまざまなイベントに参加し、多くを経験した一年であった。

# 情報処理部と各種イベント
情報科学科の高校に入学後、私は情報処理部に入部した。当初はゲーム制作を行うことを想定していたが、10月頃まで学校で使用可能な情報機器が配布されておらず（スマートフォンは校内での使用が禁止されていた）、実際の開発活動を行うことが難しい状況であった。そのため、この期間はITパスポート試験に向けた基礎学習に取り組むこととした。
また、当時の先輩にはDocker環境の構築やCTF用サーバの構築などを行っている者や、高度情報技術者試験に挑戦している者もおり、技術的に非常に刺激を受ける環境であった。これらの先輩の活動は、情報技術分野に対する自分の関心をより強める契機となった。
また、さまざまな情報に関するイベントへ参加した。大学生と共同で行うサイバーセキュリティーの対策に関するハッカソンである、「anti-cyber-crime-caffe」や、高校と提携している専門学校による、夏期集中講義、理化学研究所で開催される、スーパーコンピュータの富岳の見学などである。これらの活動でも、幅広い実践的な知見を得られた。

# IT資格について
夏休みに入ってしばらく経った頃、本来は翌年3月に受験する予定であったITパスポート試験について、現在の自分の知識でも十分に対応できると判断し、受験時期を半年早めることを決定した。
学習方法としては、まずITパスポート試験用の教科書の内容をノートに整理してまとめ、その後「過去問道場」などの問題演習サイトを用いて過去問題を繰り返し解くという方法を採用した。その結果、10月に受験した試験では600点台で合格することができた。周囲から評価を受けたことは当時大きな自信につながった。
その後、基本情報技術者試験に向けた学習を開始し、翌年2月に合格した。また、その他の資格として全商ビジネス文書実務検定を受験した。3級は授業の一環として受験し、その内容から1級にも対応できると判断したため挑戦した。1回目の受験では筆記試験で不合格となったが、2回目の受験で合格することができた。

# シンギュラリティバトルクエスト
情報処理部に入部した当初は、学校の設備が整っていなかったこともあり、具体的な活動は多くなかった。部活動の時間は主に友人と技術の話をする程度であった。
そのような状況の中、先輩から「シンギュラリティバトルクエスト」という大会への参加を勧められたため、参加することにした。この大会には複数の競技種目があり、私はセキュリティ分野のCTF（Capture The Flag）競技に参加した。
それまで自分はパソコン技術にある程度自信を持っていたが、実際の問題は想像していたよりもはるかに難しく、多くの場面で苦戦することとなった。しかし、二次選考の結果として、所属チームは決勝に進出することができた。
大会本番では序盤は比較的上位の順位を維持していたものの、後半になるにつれて難易度の高い問題が残り、解答が進まなくなった。その結果、最終順位は参加チームのほぼ中間程度となり、決して満足できる結果ではなかった。しかし、情報分野の大会に初めて参加し、自分の技術力の位置を客観的に認識できたことは非常に重要な経験であった。

# パソコンを組み立てる
高校入学後しばらくして、中学3年生の頃に購入してもらったノートPCが突然故障した。親戚に確認してもらったところ、マザーボードの故障である可能性が高いと判断された。保証にも加入していなかったため、修理は断念せざるを得なかった。
しかし、情報科学科に在籍している以上、自分専用のパソコンがない状況は望ましくないと考えた。そこで保護者と相談し、学習目的も兼ねて自分でパーツ構成を考え、総額約7万円で自作PCを組み立てることにした。
結果としてこのPCは問題なく動作し、現在も継続して使用している。専用のグラフィックボードを搭載していない構成ではあったが、軽量なゲームであれば問題なく動作する程度の性能を確保することができた。

# 暗記用のウェブアプリ制作
学校の定期試験の多くは暗記中心の内容であった。そのため、効率的に暗記学習を行うためのツールとして、問題を事前に登録し、それを解答して自己採点できるウェブアプリケーションを制作し。
実は中学生の頃、Unityを用いて類似した暗記アプリを制作した経験があった。しかし今回は、場所を問わず利用できるようにすることを目的としてWebアプリケーションとして実装することにした。
当時の自分はHTML、CSS、JavaScriptの基本的な知識しか持っていなかったが、インターネット上の情報を調査しながら開発を進めた。サーバーサイドはPHPで実装し、開発環境としてXAMPPを用いた比較的伝統的な構成を採用した。
最終的にアプリケーションの完成およびレンタルサーバーへのデプロイまで行うことができた。しかし、実際には利用者がほとんどおらず、実用的に使われることはあまりなかった。

# 一年生のまとめ
高校1年次は、長く志望していた高校に進学できたこともあり、さまざまなことに挑戦した一年であった。同時に、技術的にも大きく成長した時期であったと言える。
この一年で身についた主な内容としては、Web開発の基礎、C言語の基礎、セキュリティ分野の基礎知識、さらに情報およびビジネス分野に関する基本的な知識などが挙げられる。
一方で、この頃から徐々にゲーム制作への関心が薄れ始め、情報分野の中で自分が何を専門として取り組むべきかが分からなくなっていた時期でもあった。つまり、情報分野を志したものの、具体的にどの方向へ進むべきかを模索していた段階であったと言える。
