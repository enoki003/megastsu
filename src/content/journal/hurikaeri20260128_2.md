# はじめに
前回に引き続き、学業面での振り返りを行う。今回は高校2年生の内容を中心に扱い、1年生の頃と比べてどのように変化したかについても記述する。
また、ここではあくまで経験の振り返りを目的としている。そのため、個々の内容には軽く触れるにとどめ、深くは立ち入らない。各テーマについては、いずれ別に書くと思う。

# ディープラーニングとの出会い
年の初め、何をやればよいのかわからなかった私は、本屋のPC書籍コーナーでオライリーの『ゼロから始めるディープラーニング』を購入した。ちょうどLLMの精度が向上しつつあった時期であり、AIの仕組みを理解するべきだと考えていたからである。

本を読み進めるうちに、指数関数や総和記号、偏微分、行列など、高校1年生を終えたばかりの段階では知らない数式が数多く出てきた。これをきっかけに、数学をきちんと学ばなければならないという意識が芽生えた。このとき、まずは別の分野を先に理解してからこの本に戻ろうと考え、逆誤差伝播法あたりで一度中断することにした。

# データサイエンス
友人がデータサイエンスの道を志し、志望大学を定めた。コンピュータの扱いに慣れており、初歩的な知識の基盤があった私は、まったくの初心者であるその友人が希望する進路に進めるよう、データサイエンスの勉強を始めた。

というのも、私の学校は専門学科であるため、普通科の授業が大幅に削減されており、大学進学を目指す場合、総合型選抜や推薦系入試でなければ厳しい状況だったからである。

私は、タイタニックの生存者予測をランダムフォレストで行うチュートリアルを通じて、データの前処理、特徴量エンジニアリング、モデルの学習、評価といった一連の流れを経験した。ここで、データサイエンス（統計学）と、深層学習を含む機械学習の分野が密接に関係していることを認識した。

その後、KaggleやSIGNATEの初心者向けコンペに参加し、実践的なスキルを学んでいった。もっとも、KaggleやSIGNATEが実践的かどうかについては議論の余地がある。

# 微分積分・線形代数
データサイエンスや深層学習を進める中で、改めて重要だと認識したのは、微分積分、線形代数、統計学といった数学である。そこで私はすぐに古本屋に向かい、『微分積分』『線形代数』『統計学』の本を購入した。

まだ数学II・Bすら終えていない段階で、微分積分、線形代数、統計学の内容を理解するのは非常に難しかった。しかし、それぞれが何を扱う学問であり、何がうれしいのかは少しずつ理解できた。また、こうした分野を学ぶうえでは、高校までの数学がやはり重要であると認識した。

# BERTによるツイートからのうつ病推定
夏休みの保健体育の課題として、保健に関する内容を調べ、レポートを書いて発表するというものがあった。私は情報科学科らしい内容にしたいと思い、若者の死因の上位に自殺が含まれるという、以前に聞いた情報をもとに題材を選んだ。

文献を調べるうちに、人が自殺へと至る過程の最終段階として、うつ状態が存在することを知った。そこで、日常的にSNSを利用し、自らの私生活を発信している人が多いことから、その発信内容からうつ状態を推定できれば、うつ状態にある人を早期に発見でき、結果的に自殺を減らせるのではないかという仮説を立てた。

そこで、Googleが開発したモデルであるBERT（Bidirectional Encoder Representations from Transformers）と、分散型SNSであるMisskeyのAPIを利用してツイートデータを収集し、プロフィール情報からラベルを作成したうえで、文章からうつ病である確率を予測する分類モデルを作成した。

ただし、この方法には倫理的・方法論的な問題がある。第一に、うつ病かどうかを判断するうえで、文章が有効な指標であるという実証が十分に得られていない。第二に、プロフィールでうつ病を公言している人が、実際にうつ病でない可能性もある。また、うつ病でないとラベル付けしたツイートの中に、実際にはうつ病である人が含まれている可能性もある。

つまり、課題のために暫定的な分類を行ったにすぎず、この研究を正確に進めるには、前提となる研究の整理、倫理的論点の検討、複数の因果の考慮などが必要である。それも踏まえ、発表では問題点についても説明した。この探究を通じて、背景→仮説→方法→検証→考察という科学的なプロセスを学び、あわせて倫理上の問題についても考えることができたと思う。

# 情報処理競技大会
私が所属している情報処理部では、毎年、全国商業高等学校協会が主催する情報処理競技大会がある。この大会では、関連知識、データベース、SQLの知識が問われるテストと、プログラミング・表計算の知識が問われる筆記テストを受け、総合点を競う。内容は全商情報処理検定に準拠している。

この大会に向けて、私は部活動の時間に過去問を解き、部員と一緒にわからないところを教え合っていた。そして部内予選で先輩を破り、地域予選に参加した。

しかし、部内では1位だったにもかかわらず、本番では思うような結果を出せなかった。団体では優勝したが、個人では他校の生徒や友人に点数で上回られ、3位という結果に終わった。それでも、全国大会には主要メンバーとして参加することができた。

大会メンバーで教え合いながら準備を進め、迎えた当日に試験を受けた。結果は時間の都合上その場では確認できなかった。後から聞いたところ、私のチームは47チーム中31位だったそうだ。情報の知識については自信があったが、このような大会に参加すると、自分の想像以上に世の中は広いのだと実感した。

# 重力四目並べ大会
私の学校では、Javaの授業の集大成として、チームに分かれて他校を招き、重力四目並べのBotを作成して対戦する大会がある。

チーム競技ではあるものの、個人の能力差が大きく、また戦力バランスの関係から、技術力が比較的高い人は分散されていた。そのため、チームに分かれてはいたものの、実際にはほとんど自分が作業することになっていた。

この大会で私が作成したアルゴリズムはMini-Max法をベースとし、最適化のためにαβ枝刈りと、探索結果を一時的にキャッシュする仕組みを用いた。これにより、ターンが進むごとにより速く、より広く探索できる設計にした。また、報酬関数については論文を読んだり、重みを調整したりして、最終的には人間ではほとんど勝てないようなBotに仕上がった（ゼロサムゲームであるため、手番によっては負ける）。

大会当日、順調に勝ち進み、3位決定戦まで進んだが、そこで敗れ、4位という結果に終わった。総当たり戦ではないとはいえ、3位を逃したことは大きなショックだった。

おそらく、最終調整として加えた「勝利までのステップ数が少ないほど高い報酬を与える」というアルゴリズムが裏目に出たのだろう。また、報酬関数を複雑化しすぎたことで、選択の根拠がブラックボックス化してしまっていた面もあったと思う。

この大会では、強化学習やAlphaGoのようなゲーム理論的アルゴリズムについて学んだだけでなく、チームのまとめ方や透明性の重要性についても学ぶことになった。

# セキュリティ大会
私の学校では、毎年2年生の間でCTF大会が開催される。そこでは班をランダムに振り分けられ、3年生と企業の方々が用意した問題を解いて得点を競う。私は個人4位という結果に終わった。

CTFは本来チーム競技であるため、私はチーム内の能力を把握し、それぞれに問題を割り振る管理的な役割を担った。しかし、コンピュータに不慣れな者が多かったため、比較的簡単だと判断した問題を割り振り、技術力が比較的高かった私は高得点かつ高難度の問題を担当した。

結果として、高難度の問題も一部は解けたが、断念したものもあった。一方で、チームメイトはそもそもコンピュータに不慣れであり、得点を獲得できなかった。

そして、この大会で上位3位に入った人物の戦略を分析すると、協力やチームでの連携をほとんど行わず、自分が解ける問題をひたすら解いていた。そもそもチーム競技では、チームの技術レベルがある程度均等であることが重要であり、今回の大会ではチーム分けがランダムだったため、前提条件が不均衡だったと言える。

したがって、この場合は時間制約が大きく、チームにやり方を教えたり分担を考えたりする時間が大きなロスになる。ゆえに、最適な戦略は、各自が自分で解けそうな問題を優先して解いていくことだった。私はそれまで「チーム競技では協力すべきだ」と考えていたが、必ずしもそれが最適解ではないことを知った。

# 資格の数々、応用情報への挑戦
資格に関しては順調だった。全商の情報処理検定では2部門で1級を取得し、11月には情報セキュリティマネジメント試験にも合格した。

その後、秋期の応用情報技術者試験に向けて日々対策を進めた。試験当日には対策してきた力を発揮したつもりだったが、午前試験で残り4点足りず、不合格となった。大会やテストの時期が重なっていたこともあるが、過去問演習の不足が大きかったと思う。

そこで、翌年の春期試験に向けて再び勉強した。その際には前回の反省を踏まえ、とにかく多くの過去問を解き、予想問題集にも取り組んだ。試験当日は、私の得意分野であるAIやデータサイエンス系統の問題が多く出題され、自己採点では6割を超えていたが、結果発表ではまたしても4点足りなかった。

金銭面の問題もあったため、応用情報技術者試験はいったん断念し、合格のために何が必要かを分析した。そこで、私は資格本から知識を得ていたものの、それが実際にどういうもので、どこで使われるのかという理解が不足していたことに気づいた。そして、計算機科学の基礎であり、より古典的な分野から改めてしっかり学び直すことにした。

# シンギュラリティバトルクエスト優勝
2025年度のシンギュラリティバトルクエストのデータサイエンス部門（データクエスト）に、友人と参加した。理由は、友人の志望校への進学を手助けしたかったことと、自分の能力を試したかったからである。

地区予選は統計学に関する問題であり、これはJMOOCである程度勉強していたため、難なく突破できた。二次選考では、日本の高齢化率の予測を行った。与えられたデータは年ごとの年齢階級別人口に関するものだったため、時系列予測モデルを使うことにした。単回帰分析とARIMAモデルによる予測の二つのグラフを作成し、NotebookLMに結果をまとめ、審査員の質問にも答えて二次選考を通過できた。

最終選考では、夜間光データ（リモートセンシングデータ）を用いた分析が課題として提示された。私は「分散型都市の形成」をテーマに設定した。地域の分散のためには、経済的資本と技術力をもつ人材を誘致する必要があると考え、経済力と技術力の双方を持ち、かつ働きながら旅行をするデジタルノマドに着目した。そして、デジタルノマドを誘致するためには、各地域のインフラ整備状況を示す独自指標が必要だと考え、政府統計を用いた指標を作成し、夜間光との因果や相関を分析した。

最終的には、夜間光からその指標を推定するモデルを作成し、プレゼンテーションを行った。文献に基づいた分析、因果推論や統計的手法の活用、プレゼンテーションの質、NotebookLMの整理の丁寧さが評価され、優勝することができた。

初めて地理データの分析に取り組んだこともあり苦戦したが、最終的に優勝できたことで大きな自信を得た。また、「いかに妥当性を持たせるか」を試行錯誤する中で、科学的な方法について強く意識するようになり、レポートも概要、背景、仮説、結果、考察の順序で書くようになった。ジョン・デューイの『How We Think』にも大きな影響を受けた。


# 2年生まとめ
2年生になってからは、機械学習とデータサイエンスを中心に勉強を進めるようになり、関心の方向性はより理論寄りになっていった。もっとも、機械学習は一般に工学寄りの分野であるが、私の関心は制度よりも、その理論的な方法論の部分にあった。

また、データサイエンスのように透明性が問われる分野や、機械学習のように具体的な評価指標が問われる分野に触れたことで、科学的な方法論や妥当性についての感覚を身につけることができた。

一方で、応用情報技術者試験に不合格だったことからもわかるように、自分の無知を知り、より厳密で正確な知識を得ようと努力し始めるきっかけにもなった。
