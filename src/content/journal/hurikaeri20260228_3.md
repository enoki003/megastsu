# はじめに
前回に引き続き高校生活を振り返る。今回が最後の記事であり、内容は高校三年生の経験についてである。


# 「シンギュラリティはより近く」との出会い
２年生の時にシンギュラリティバトルクエストの副賞として頂いた、レイ・カーツワイル著の『シンギュラリティはより近く』をいただいた。少しSFのようではあるが、カーツワイルのBMIと人間の脳が接続され、想像のつかないものになるという考えや、それに関係する哲学や社会科学の話は情報という分野であるのにも関わらず、多岐にわたる分野の幅広い知識が登場し驚いた。ユヴァル・ノア・ハラリの『ホモ・デウス』をきっかけに、読書を積極的に行うようになった。

# 課題研究
一年のうちに関心や仮説が大きく変化したため、結果として研究テーマを転々と変更してしまった。この点については、問題設定と研究計画の設計が不十分であったと認識している。
一番最初は鳥の脳から発想を得た、スパースなニューラルネットワークの開発をテーマに事前に調べていた。ただ、それをするのには神経科学も必要で、ましてや鳥の脳なんてなかなか先行研究がなかった（もちろん論文や計算論的神経科学のテキストを読んではいた）。また、タスクを事前に設定していないことを先生に指摘され、評価指標が定義されていない状態で理論だけが先行していた。この状態では研究として成立しないと判断し、一旦この研究を断念して、LLMのエージェント関連の研究に移行した。私はLLMが計算資源を前提にしていることに疑問を抱き、1Bレベルの小型のLLMを組み合わせてより高度な出力を得られないかを次のテーマとして探求をした。今でいう、clodeCodeのオーケストレーションのような発想であった。黒板型アーキテクチャを採用し、役割分担や再帰的な呼び出しを可能にしたが、小型モデルであるがゆえに柔軟な出力が得られず、結果として失敗に終わった。
時間も計算資源もこのままではまずいと思ったため、方向を転換した。その内容は、群知能アルゴリズムであるBoidsに類似した手法を用いて、MoE（Mixture of Experts）の専門家ルーティングを行うというものであった。しかしこれも間に合わないと感じたため方向を変え、最終的には画像認識をタスクとし、同じくMoEに近いルーティングによってモデルを選択する方法を考えた。具体的には、専門家同士の関係を示す行列と粒子群最適化アルゴリズムを組み合わせたルーティング手法を採用した。比較実験が難しい分野でもあり、評価には苦戦したがなんとか終えることができた。
仮説に対して期待した結果が出なかったり、いかに伝わるように発表するのかなど色々と試行錯誤した経験を得ることができた。

# 哲学
きっかけの詳細は割愛するが、哲学に興味を持ち、様々な書籍を読んだ。
デカルト、プラトン、アリストテレスなどのメジャーなものから、オルテガやジョン・デューイ、構造主義やカミュなど幅広く学んだ。哲学書を読む中で、前提や定義そのものを疑う姿勢が強まり、思考対象を一段抽象化して捉えるメタ的な思考が可能になったと認識している。同時に、学問分野同士の有機的な連関にも気づき始めた。
この頃から友人とよく社会や人生について語るようになった。そこでさらに自身の思考を矯正したり、深くすることができたと思う。これにより、対象の範囲を広く捉えつつ、命題と現実の対応関係を意識し、主観的バイアスを抑制した思考を志向するようになった。私がより幅広くかつ、より正確にかつ、より客観的であろうとするようになったきっかけにもなった。どのように思考が変化し、どうなったのかについてはまた別の記事にて解説する。


# AOに提出した作品 「DewAI」
大学受験を総合選抜で受けることに決めた。理由は、経験と資格が多く有利に働くと判断したこと、また専門学科であるため一般入試では不利であったためである。
最初は課題研究の内容を提出しようとしていたが、課題研究のテーマ変更や難航もあり、AOだけのアプリを作成することに決めた。私がこれまでに学んだこと、私のやりたいこと、私自身の特性を反映できるようなものを考えた結果、ローカルで動作するLLMに対して、発話の一貫性や論理的整合性に基づくペナルティを付与し、指定した人数で議論を行えるアプリを開発することにした。
目的は、エコーチェンバー現象の抑制を前提として、議論における視点の多様性と論理的一貫性を維持しながら思考を深化させることである。技術スタックとしては、Web技術でネイティブアプリを構築可能なフレームワークであるTauriを、ReactおよびTypeScriptと組み合わせて使用した。また、説明書とDockerイメージの構築もした。
面接では作った経緯や指摘、これまでの経験について問われ、最終的に合格することができた。

# デパートのアンケートシステムの共同開発
私の高校で行われる販売学習で、顧客からアンケートを受けてもらい、その回答を店舗に置いた大きなディスプレイに表示するアプリを３人で共同開発した。設計はバックエンド、管理者画面、回答表示画面、アンケート画面に分割して行い、私はアンケート画面を担当した。また、回答表示画面に関してはデザインを迷っていたため、各自一つ作って良い方を採用する方針をとった。先生から簡潔な要件定義を言い渡され、納期が４日だったため、全員がほぼバイブコーディングで作業したものの、一定の品質を担保した成果物を完成させることができた。当日は無事に動作し、モニターやPC・Wifiのセットアップまで共同で行い、バックアップまで行うことができた。ほとんど初めての共同開発で、責務分離や設計、Githubでの共同作業になれる良い経験となった。この時のことについてもまた別の記事で書こうと思う。

# 終わりに
3年生では理学や情報だけでなく、たくさんの学問に触れ、その大切さを知った１年であった。ここで書いた以外にも、物理で微積を用いて公式を導いたり、日本語そのものについて勉強したり、多岐にわたる分野に興味を持ち勉強していた。その結果、内面的に大きく成長したと考えている。また、課題研究のように懐疑が強く出てしまった結果、一貫して研究を行うことができなかったことは心残りである。時間がないと言い訳してバイブコーディングに頼り切りでもあった。そこは反省して、計算機科学の原点から厳密に復習し直そうと考えている。
また、ここの三年間で興味・関心が情報だけから人工知能と情報を主軸にそれ以外の学問にも興味・関心を持つようになった。中学の頃は「使えない学問はやる意味がない」、「理論をこねるよりも何かを作った方ががいい」などといかにも合理主義的な考え方であったが、高校三年間の様々な経験を通して、対象となる事象のイデアから逆算して思考し、真理は有機的かつ多元的であり単純な二項対立では表現できないと理解したことで、結果や経験に依存せず、前提から演繹的に思考する、より理論的な思考様式へと変化したと考えている。
もちろん、これが以前の合理主義と比較して優れているかについては一概には判断できない。弁論上必要であるため二項対立的な表現を用いるが、少なくとも、二項対立では記述できない問題や未確定な命題を扱えるという点において、従来の単純化された合理主義よりも拡張された思考へと移行したと考えている。
しかし、現在の思考にも依然として懐疑すべき点は存在し、過去の自分と同様に、将来の自分が異なる思考様式に至る可能性もある。むしろ、それを期待して、大学以降もしっかりと学んでいこうと思う。
